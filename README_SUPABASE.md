# Supabase 数据库集成说明

## 配置步骤

1. **环境变量配置**
   - 复制 `.env.example` 文件为 `.env`
   - 在 `.env` 文件中设置你的 Supabase 匿名密钥：
     ```
     VITE_SUPABASE_KEY=your-actual-supabase-anon-key
     ```

2. **数据库表结构**
   数据库名：`supabase-daka`
   表名：`user`
   
   ```sql
   create table public.user (
     id bigint generated by default as identity not null,
     nick_name character varying null,
     name character varying null,
     daka_result character varying null,
     ip character varying null,
     ua character varying null,
     created_at timestamp with time zone not null default now(),
     phone character varying null,
     team_name character varying null,
     constraint user_pkey primary key (id)
   ) TABLESPACE pg_default;
   ```

   自动打卡配置同步表：

   ```sql
   create table public.daka_settings (
     account_no text not null,
     schedule_config jsonb not null default '{}'::jsonb,
     daka_config jsonb not null default '{}'::jsonb,
     updated_at timestamp with time zone not null default now(),
     constraint daka_settings_pkey primary key (account_no)
   ) TABLESPACE pg_default;
   ```

3. **Supabase 权限配置**
   - 启用匿名用户登录：在 Supabase 项目设置 > Authentication > Settings 中启用 "Enable anonymous sign-ins"
   - 配置 RLS 策略：确保匿名用户可以向 user 表插入数据

4. **功能说明**
   - 每次记录数据前会自动创建匿名用户身份，防止401权限错误
   - 用户登录成功时会记录用户信息到数据库
   - 打卡成功、失败或异常时都会记录相应的状态
   - 自动获取用户IP地址和浏览器UA信息
   - 登录成功后会使用海康 `accountNo` 从 `daka_settings` 同步自动打卡和地点配置

5. **部署到 Vercel**
   - 在 Vercel 项目设置中添加环境变量 `VITE_SUPABASE_KEY`
   - 确保 Supabase 项目启用了匿名登录功能

## 数据记录时机

- **登录成功**: `daka_result = 'login_success'`
- **打卡成功**: `daka_result = 'daka_success'`
- **打卡失败**: `daka_result = 'daka_failed: 错误信息'`
- **打卡异常**: `daka_result = 'daka_error: 异常信息'`

## 权限处理

应用使用匿名用户身份进行数据库操作，每次调用 `recordUserInfo` 函数时会：
1. 自动创建匿名用户会话
2. 使用该会话身份插入数据
3. 即使匿名登录失败也会尝试插入数据（向后兼容）
